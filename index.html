Useful Kubernetes Commands
Essential commands for managing your Kubernetes cluster.
Cluster management commands
# View cluster nodes
kubectl get nodes -o wide

# View all pods in all namespaces
kubectl get pods --all-namespaces

# View services
kubectl get services

# View deployments
kubectl get deployments

# Describe a specific pod
kubectl describe pod <pod-name>

# View pod logs
kubectl logs <pod-name>

# Execute commands in a pod
kubectl exec -it <pod-name> -- /bin/bash
Troubleshooting commands
# Check cluster events
kubectl get events --sort-by=.metadata.creationTimestamp

# Check node resources
kubectl top nodes

# Check pod resources
kubectl top pods

# Reset cluster (if needed)
sudo kubeadm reset
sudo rm -rf /etc/cni/net.d
sudo rm -rf $HOME/.kube/config

Install Docker on Kali
Set up Docker environment with all necessary components for containerization.
Update system and install Docker
# Update system
sudo apt update && sudo apt upgrade -y

# Install Docker
sudo apt install -y docker.io

# Start Docker service
sudo systemctl start docker
sudo systemctl enable docker

# Verify installation
docker --version
Optional: Add your user to Docker group to avoid using sudo:
Configure Docker user permissions
sudo usermod -aG docker $USER
newgrp docker

Install Kubernetes (Minikube + kubectl)
Set up local Kubernetes environment using Minikube for development and testing.
Install kubectl
# Install kubectl
curl -LO "https://dl.k8s.io/release/$(curl -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
chmod +x kubectl
sudo mv kubectl /usr/local/bin/

# Verify installation
kubectl version --client
Install and start Minikube
# Install Minikube
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube

# Start Minikube using Docker driver
minikube start --driver=docker
4
Dockerize Laravel Project
Create Docker configuration files for your Laravel application.
Create Dockerfile in project root
FROM php:8.2-fpm

# Install system dependencies
RUN apt-get update && apt-get install -y \
    git curl libpng-dev libjpeg-dev libfreetype6-dev zip unzip

# Install PHP extensions
RUN docker-php-ext-install pdo pdo_mysql gd

# Install Composer
COPY --from=composer:2 /usr/bin/composer /usr/bin/composer

# Set working directory
WORKDIR /var/www

COPY . .

RUN composer install --no-dev --optimize-autoloader

CMD ["php-fpm"]

EXPOSE 9000
Create docker-compose.yml
version: '3.8'
services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    volumes:
      - .:/var/www
    depends_on:
      - db
    networks:
      - laravel

  db:
    image: mysql:8
    environment:
      MYSQL_DATABASE: laravel
      MYSQL_ROOT_PASSWORD: root
      MYSQL_USER: laravel
      MYSQL_PASSWORD: secret
    ports:
      - "3306:3306"
    networks:
      - laravel

  nginx:
    image: nginx:alpine
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
      - .:/var/www
    ports:
      - "8000:80"
    depends_on:
      - app
    networks:
      - laravel

networks:
  laravel:
Create nginx.conf
server {
    listen 80;
    index index.php index.html;
    root /var/www/public;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_pass app:9000;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
    }
}
Run Docker Compose
docker-compose up -d
üéâ Laravel should now be available at: http://localhost:8000
5
Deploy Laravel on Kubernetes
Convert Docker Compose configuration to Kubernetes manifests and deploy.
Install kompose converter
sudo apt install -y kompose
Convert and deploy to Kubernetes
# Convert Docker Compose to Kubernetes manifests
kompose convert -f docker-compose.yml

# Apply Kubernetes configurations
kubectl apply -f .
Expose service and access application
# Expose nginx service
kubectl expose deployment nginx --type=NodePort --port=80

# Open application in browser
minikube service nginx
üöÄ Congratulations! Laravel is now running inside a Kubernetes cluster with Docker + Kubernetes deployment on Kali Linux.

Install Required Packages
Install essential packages needed for Kubernetes repository configuration.
Install required dependencies
sudo apt install -y apt-transport-https ca-certificates curl
3
Add Kubernetes Repository
Configure the official Kubernetes repository for package installation.
Add GPG key and repository
sudo curl -fsSLo /usr/share/keyrings/kubernetes-archive-keyring.gpg https://packages.cloud.google.com/apt/doc/apt-key.gpg

echo "deb [signed-by=/usr/share/keyrings/kubernetes-archive-keyring.gpg] \
https://apt.kubernetes.io/ kubernetes-xenial main" | \
sudo tee /etc/apt/sources.list.d/kubernetes.list
Update repository
sudo apt update
4
Install Kubernetes Components
Install kubelet, kubeadm, and kubectl - the core Kubernetes components.
Install and hold Kubernetes packages
sudo apt install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
Note: The 'hold' command prevents automatic updates that could break your cluster.
5
Disable Swap
Kubernetes requires swap to be disabled for proper operation.
Disable swap permanently
sudo swapoff -a
sudo sed -i '/ swap / s/^\(.*\)$/#\1/' /etc/fstab
6
Enable & Start kubelet
Configure kubelet service to start automatically on boot.
Start kubelet service
sudo systemctl enable kubelet
sudo systemctl start kubelet
7
Initialize Kubernetes Cluster
Initialize the master node with a pod network CIDR for communication.
Initialize master node
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
After successful initialization, copy the kubeadm join command shown for adding worker nodes.
