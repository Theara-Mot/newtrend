2‚öôÔ∏è Common Setup (All Nodes)

Run these commands on all three servers:

# Ubuntu / Kali
sudo apt update -y && sudo apt upgrade -y

# CentOS
sudo yum update -y
# Disable swap (required for Kubernetes)
sudo swapoff -a
sudo sed -i '/ swap / s/^/#/' /etc/fstab
# Enable required kernel modules
sudo modprobe br_netfilter
echo '1' | sudo tee /proc/sys/net/bridge/bridge-nf-call-iptables

3üê≥ Install Docker (Container Runtime)

Ubuntu / Kali Installation:

sudo apt install -y apt-transport-https ca-certificates curl gnupg lsb-release

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker.gpg

echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
$(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io

sudo systemctl enable --now docker

CentOS Installation:

sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io

sudo systemctl enable --now docker
üí° Optional: Add your user to docker group to run docker without sudo:
sudo usermod -aG docker $USER
Logout and login again for changes to take effect

sudo ufw enable

sudo ufw allow 10250/tcp        # Kubelet API
sudo ufw allow 30000:32767/tcp  # NodePort Services

6üëë Initialize Kubernetes Master

Run on CentOS Master (192.168.56.148):

sudo kubeadm init --apiserver-advertise-address=192.168.56.148 --pod-network-cidr=10.244.0.0/16
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
kubectl apply -f https://raw.githubusercontent.com/flannel-io/flannel/master/Documentation/kube-flannel.yml
‚ö†Ô∏è Important: Save the join command output from kubeadm init! You'll need it for worker nodes.

7üë∑ Join Worker Nodes

Run on both Ubuntu and Kali workers:

  --discovery-token-ca-cert-hash sha256:<HASH>

Label Nodes by Role (on Master):

kubectl label node ubuntu-db role=db
kubectl label node kali-app role=app

8üöÄ Deploy Applications

Database Deployment (MySQL on Ubuntu):

apiVersion: apps/v1
kind: Deployment
metadata:
  name: mydb
spec:
  replicas: 1
  selector:
    matchLabels:
      app: mydb
  template:
    metadata:
      labels:
        app: mydb
    spec:
      nodeSelector:
        role: db
      containers:
      - name: mysql
        image: mysql:8
        env:
        - name: MYSQL_ROOT_PASSWORD
          value: "rootpass"
        ports:
        - containerPort: 3306
---
apiVersion: v1
kind: Service
metadata:
  name: mydb
spec:
  clusterIP: None   # headless service for DB
  ports:
  - port: 3306
    targetPort: 3306
  selector:
    app: mydb

Application Deployment (on Kali):

apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 2
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      nodeSelector:
        role: app
      containers:
      - name: app
        image: myrepo/myapp:latest   # replace with your app image
        ports:
        - containerPort: 80
        env:
        - name: DB_HOST
          value: "mydb.default.svc.cluster.local"
        - name: DB_USER
          value: "root"
        - name: DB_PASS
          value: "rootpass"
---
apiVersion: v1
kind: Service
metadata:
  name: myapp
spec:
  type: NodePort
  ports:
  - port: 80
    targetPort: 80
    nodePort: 30080
  selector:
    app: myapp
kubectl apply -f mysql-db.yaml
kubectl apply -f myapp.yaml
